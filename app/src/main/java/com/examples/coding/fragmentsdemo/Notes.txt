- Fragments
    - What is a fragment?
        - An UI Element which acts like a sub activity
        - Fragments are hosted by activities
        - Activity is Required for a fragment to exist
        - Fragments have their own lifecycle which is tied to the activity lifecycle
            - States of Fragment Lifecycle
                - onAttach
                    - When the fragment is being assigned to the activity
                - onCreate
                    - When the fragment is inflated into memory
                - onCreateView
                    - When the view for the fragment is rendered
                - onActivityCreated
                    - After the host activity is completed with its creation
                - onStart
                    - When the fragment is coming into the foreground, visible but not interactable
                - onResume
                    - When the fragment is fully into the foreground
                - onPause
                    - When the fragment is going into the background, visible but not interactable
                - onStop
                    - When the fragment is fully into the background
                - onDestroyView
                    - When the view for the fragment is destroyed
                - onDestroy
                    - When the memory for the fragment is deallocated
                - onDetach
                    - When the Fragment is disassociated with the activity

            - 2 types of Fragments
                - static
                    - Add to activity using the <fragment> tag in the layout for the host activity
                    - Can not be removed or replaced
                - dynamic
                    - Add to the activity by using the fragment manager
                        - Begin fragment transaction
                        - 2 possible methods
                            -add
                                - adds new instance of the fragment on top of the fragment stack
                            -replace
                                - replace the current fragment located in the defined space with
                                    a new instance of the fragment
                        - MAY want to define the TAG for the FRAGMENT in the Fragment Stack
                        - commit()
